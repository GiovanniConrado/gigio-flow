# Codex for Open Source Application Notes

This document keeps the repository narrative ready for the Codex for Open Source application.

Official program page: <https://openai.com/form/codex-for-oss>

## Repository Qualification Pitch

Gigio Flow is an open-source local workspace protocol and Studio for Codex-powered AI squads. It helps maintainers preserve project context, transform product discovery into small issues, coordinate PM/CTO/Dev/QA agent roles, and connect delivery through Linear, GitHub, and Vercel.

## API Credits Usage

API credits would be used to power open-source maintainer workflows: PRD refinement, issue slicing, QA review, release note generation, security-oriented code review, and automation that keeps Markdown project memory synchronized with Linear, GitHub, and Vercel.

## Maintainer Role

Primary maintainer.

## Why It Matters

Open-source maintainers increasingly use AI coding agents, but most projects lack durable context, scoped handoffs, and auditable product decisions. Gigio Flow gives Codex a structured operating system for real maintenance work.

## Demo Story

1. A maintainer writes a raw idea.
2. Gigio Flow turns it into a product artifact.
3. Codex refines it into a small issue.
4. The Studio creates a Linear task.
5. Codex implements through GitHub.
6. Vercel preview validates the result.
7. Gigio Flow updates project memory.
